基本介紹

程式腳本的每一句指令都可以稱為敘述句（Statement），用來驅使電腦執行我們寫下的規則。一句完整的敘述句會以分號 ; 作為結尾，向 JavaScript 表示一段可執行運算的程式。根據這個定義，我們回首看看上一篇是如何宣告變數的：

var price = 20;

變數宣告後以 ; 分號結尾，沒錯，宣告變數的行為就是一句敘述句。當然啦，不是所有敘述句都這麼單純，不同的需求、不同的邏輯，敘述句的複雜程度也不一樣，有時甚至會結合多個敘述句使其成為一段複合敘述句，而通常這些敘述句逐句讀下來可以發現彼此互有關連，那麼我們就需要將它們群組起來以便識別，這個群組稱為「程式區塊」（Code Block）。

範例：

function demo() {
    var name = 'iTrong';
    console.log(name);
}

由一對大括號（或稱花括號）{ } 標示的區塊即為程式區塊，一個程式區塊內可包含一或多個敘述句。此外，程式區塊結尾不需要加上分號。

認知 ; 與 { } 在程式碼代表的意義後，接下來學習各種類型的敘述句表現手法，應該會相對容易閱讀許多。大體而言，敘述句的種類可區分為三種：條件敘述句、迴圈敘述句與跳躍敘述句。


條件敘述句

條件敘述句的語法有兩種：if 與 switch。

if：

if 是很常見的條件判斷式敘述句，基本上會搭配 else，表示「如果符合 if 的條件則執行其指令，若不是則執行 else 裡的指令」的概念。例如：

var a = 5;
var b = 3;

if (a > b) {
    document.write("a 比 b 大");
}
else {
    document.write("a 比 b 小");
}

在上例中我們宣告了變數 a 與 b，並分別指定它們的值為 5 與 3。條件式的程式區塊判斷規則為：如果 a 大於 b，則文件寫下「a 比 b 大」的結果，否則輸出「a 比 b 小」。程式執行判斷時，會判讀小括號 ( ) 裡的條件是否與變數吻合，若相同則執行寫在它程式區塊裡的敘述句，若不相同則不執行。實際判斷結果 a 數值大於 b，所以程式執行 if 裡的指令。

See the Pen JS-learn: if...else... by Trong (@itrong) on CodePen.

儘管這個範例的判斷式乍看之下沒有問題，但其實存在很大的瑕疵──沒有考慮到兩者數值相等的可能性，假設今天我們把 a 的數值改成 3，試問會發生什麼情況？

var a = 5;
var b = 3;

a = 3;

if (a > b) {
    document.write("a 比 b 大");
}
else {
    document.write("a 比 b 小");
}

See the Pen JS-learn: if...else..._2 by Trong (@itrong) on CodePen.

答對了嗎？畫面最後顯示的結果是「a 比 b 小」。其實程式判定並沒有錯，因為條件式的規則只有定義兩種結果，要嘛 a 大於 b，要嘛「其他」，這個其他同時包含了「a 小於 b」以及「a 等於 b」兩種可能，如果我們要使條件式符合邏輯，最簡單的方法就是將 else 的敘述句結果修改為「a 不比 b 小」，或是「a 小於等於 b」，但這些都是治標不治本的處理方式（除非，你希望等於與小於的條件式是相同的），比較好的方式仍是將兩者數字相等的判斷邏輯自成一格。有些剛開始學程式的人會以為 if else 只能設定兩種條件，實則不然，if 條件式可同時設定多筆條件，初始條件使用 if，其他條件使用 else if，最終條件（或稱非以上條件的結果）使用 else。

我們可以將剛才的範例條件式改寫成如下：

if (a > b) {
    document.write("a 比 b 大");
}
else if (a = b) {
    document.write("a 等於 b");
}
else {
    document.write("a 比 b 小");
}

如此一來，當 a 與 b 數值相等的情況下進入條件式判斷，就能正確輸出「a 等於 b」的結果了。

See the Pen JS-learn: if...else..._3 by Trong (@itrong) on CodePen.


switch：

當我們使用 if 處理超過一個以上的判斷條件式，可以使用 else if，除此之外還有另一種語法，那就是 switch。switch 適用場合多在進行多項條件且規則類似的條件判斷。其語法結構為：

switch(參數){
    case '條件1':
        ...
        break;
    
    case '條件2':
        ...
        break;
    
    default:
        ...
        break;
}

每一句 case 都代表一個條件，default 則表示預設（或不符合以上條件）的條件。至於 break，它代表的是一種跳出的概念，如果程式讀取到相符的條件，就會停止讀取後方的程式碼。

假設現在要寫一個判斷三原色的條件式，我們可以這樣寫：

var color = "blue";
switch(color){
    case 'red':
        document.write("紅色");
        break;
    
    case 'green':
        document.write("綠色");
        break;

    case 'blue':
        document.write("藍色");
        break;
    
    default:
        document.write("不是三原色");
        break;
}

See the Pen JS-learn: switch by Trong (@itrong) on CodePen.

switch 與 if 條件的差別在於前者會只讀取跟參數相符條件式裡的敘述句；而後者則讀取所有條件與其敘述句，效能消耗較 switch 來得多，渲染就會比較慢。


迴圈敘述句

當我們想重複執行某件事情（敘述句）十遍，我們可能會在程式碼裡面輸入十次內容，例如：

console.log(1);
console.log(2);
console.log(3);
console.log(4);
console.log(5);
console.log(6);
console.log(7);
console.log(8);
console.log(9);
console.log(10);

雖然程式依然能跑出我們要的結果，但這種作法既浪費效能，看起來也很不專業，是那種明眼的工程師一看就會噴笑的程度。但如果使用迴圈那就不一樣了，不僅將要重複執行多次的敘述句大幅輕量化，整體程式碼也變得簡潔乾淨，雖然迴圈在資深工程師的眼裡依然是很基本的概念，但起碼菜味已經沒有那麼濃郁。JavaScript 的迴圈語法有兩種：for 與 while。
