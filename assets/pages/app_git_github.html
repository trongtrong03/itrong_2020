在上一章我們已經順利學會 Git 在本機端進行版本控制的方法，儘管已經可以對專案建立一套備份機制，但這也僅止於個人工作電腦內，一旦工作電腦損壞，所有版控都會連同專案檔案一同付諸東流。因此，除了將變更、維護後的專案定期提交到版本庫儲存之外，同時將檔案一併推送到遠端的儲存庫亦是分散式管理系統很重要的一環。有關本機端檔案推送（Push）的方式筆者將發表於下一篇學習筆記，為了讓之後遠端相關操作的練習得以更快速進行，遂決定先講講建立遠端儲存庫的方法。

建立遠端儲存庫的平台或方法不少，本篇採用的對象是 GitHub──不單單只是一個透過 Git 進行版本控制的軟體原始碼代管服務，它同時也是一個被稱為開發者作品集的開源社群平台。如果你還沒有 GitHub 帳號，趕緊去辦理帳戶吧！註冊用戶之後，便可以開始進行遠端版本儲存庫的建置了。


點選右上角工具列的「+」按鈕，選擇「New repository」


賦予此儲存庫（Repository）名稱

「Initialize this repository with a README」此欄用意在於詢問是否要於儲存庫建立之後，一併建立一個「README」的文件。


按下「Create repository」完成創建

如果沒有打勾，畫面則會是這個樣子的：

很多人看到一堆「不知所云」的程式碼就先暈了一晌，但其實上面資訊只是在告訴使用者要如何將專案推送到這裡，並提供此儲存庫遠端連結及連線金鑰。

有關 GitHub 建立遠端儲存庫的流程就寫到這裡，本篇的重點在於取得一個遠端儲存庫的連結，以方便後續我們進行本機與遠端互動的練習。
