前言

在學習網頁設計的基礎中，初學者都會被灌輸網頁是由 HTML、CSS、JavaScript 構成的說法：HTML 是骨幹、CSS 負責美化，而 JavaScript 則用來實現網頁與使用者之間的互動行為。然而深入探討後就會發現，真正實現網頁互動行為的操作方法是由 JavaScript 的執行平台──也就是「瀏覽器」所提供，它們（操作方法）基本上會由「BOM」和「DOM」兩個物件持有，前端開發者即透過 JavaScript 去呼叫 BOM 與 DOM 提供的 API 來控制瀏覽器的行為與網頁的內容。

除了 BOM 與 DOM 兩大物件模型外，JavaScript 亦內建以 ECMAScript 標準為基礎的標準物件，接下來的幾篇文章將來記錄這些物件模型與其操作方法的內容。


  • BOM（瀏覽器物件模型）
  • DOM（文件物件模型）
  • BOM 與 DOM 的差異

BOM（瀏覽器物件模型）

Browser Object Model，簡稱 BOM，中文意為瀏覽器物件模型，顧名思義是瀏覽器提供的物件，是一組與瀏覽器視窗有關的物件模型。其核心物件為 window 物件，指的是目前瀏覽器的視窗或頁籤，它在瀏覽器主要扮演著「操作全域物件（Global Object）」及「JavaScript 用來與瀏覽器溝通的窗口」兩個角色。

首先我們來談談「全域物件」的部份，全域物件（Global Object）指的是在全域作用範圍內宣告的變數、物件、函式...等，通通都會自動變成全域物件的屬性。例如：

var myName = 'itrong';

這樣以 myName 宣告的變數，我們稱之為全域變數，可以透過 window 去取得它們。例如：

console.log(window.myName);

這裡額外需要瞭解的是，在全域作用範圍內宣告的全域變數，無法直接用 delete 關鍵字移除變數，譬如：

var myName = 'itrong';
console.log(window.myName);     // "itrong"

delete window.myName;
console.log(window.myName);     // "itrong"

但如果是用 window 物件指定的屬性就能透過 delete 關鍵字移除：

window.myName = 'itrong';
console.log(window.myName);     // "itrong"

delete window.myName;
console.log(window.myName);     // undefined


至於「JavaScript 用來與瀏覽器溝通的窗口」，在初心者時期最常接觸到的莫過於「alert()」了，這個方法可以呼叫出瀏覽器內建的警告框。舉例來說：

window.alert("Hello!");

通過這樣的程式碼，瀏覽器就能彈出帶有「Hello!」訊息的警告對話框，又或者是用 window.confirm()、window.prompt()，同樣也能實現類似的功用。
