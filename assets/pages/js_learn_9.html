前言

在學習網頁設計的基礎中，初學者都會被灌輸網頁是由 HTML、CSS、JavaScript 構成的說法：HTML 是骨幹、CSS 負責美化，而 JavaScript 則用來實現網頁與使用者之間的互動行為。然而深入探討後就會發現，真正實現網頁互動行為的操作方法是由 JavaScript 的執行平台──也就是「瀏覽器」所提供，它們（操作方法）基本上會由「BOM」和「DOM」兩個物件持有，前端開發者即透過 JavaScript 去呼叫 BOM 與 DOM 提供的 API 來控制瀏覽器的行為與網頁的內容。

除了 BOM 與 DOM 兩大物件模型外，JavaScript 亦內建以 ECMAScript 標準為基礎的標準物件，接下來的幾篇文章將來記錄這些物件模型與其操作方法的內容。

BOM 簡介

Browser Object Model，簡稱 BOM，中文意為瀏覽器物件模型，顧名思義是瀏覽器提供的物件，是一組與瀏覽器視窗有關的物件模型。其核心物件為 window 物件，指的是目前瀏覽器的視窗或頁籤，它在瀏覽器主要扮演著「操作全域物件（Global Object）」及「JavaScript 用來與瀏覽器溝通的窗口」兩個角色。

首先我們來談談「全域物件」的部份，全域物件（Global Object）指的是在全域作用範圍內宣告的變數、物件、函式...等，通通都會自動變成全域物件的屬性。例如：

var myName = 'itrong';

這樣以 myName 宣告的變數，我們稱之為全域變數，可以透過 window 去取得它們。例如：

console.log(window.myName);

這裡額外需要瞭解的是，在全域作用範圍內宣告的全域變數，無法直接用 delete 關鍵字移除變數，譬如：

var myName = 'itrong';
console.log(window.myName);     // "itrong"

delete window.myName;
console.log(window.myName);     // "itrong"

但如果是用 window 物件指定的屬性就能透過 delete 關鍵字移除：

window.myName = 'itrong';
console.log(window.myName);     // "itrong"

delete window.myName;
console.log(window.myName);     // undefined


至於「JavaScript 用來與瀏覽器溝通的窗口」，在初心者時期最常接觸到的莫過於「alert()」了，這個方法可以呼叫出瀏覽器內建的警告框。舉例來說：

window.alert("Hello!");

通過這樣的程式碼，瀏覽器就能彈出帶有「Hello!」訊息的警告對話框，又或者是用 window.confirm()、window.prompt()，同樣也能實現類似的功用。

當我們使用 window 內的物件，前面的 window 是可以省略不寫的，因此我們才會看到或曾經直接使用 alert() 操作程式碼內容的寫法。


常見子物件

不管是 alert()、confirm() 還是 prompt()，它們其實都是 window 的子物件（它們既是「方法」，卻同時也是物件），window 的子物件很多，以下列舉一些常見的子物件並作簡單介紹：

  • document
  • screen
  • history
  • location
  • navigator
  • frames

document：

document 指的是視窗裡目前顯示的文件物件，它其實就是「DOM」文件物件模型，換言之，DOM 乃 BOM 核心 window 的子物件之一。所以關於 document 的詳細內容，暫且留到下一個章節再作說明。

history：

history 可以用來操作瀏覽器頁面的歷史記錄，例如常見的載入瀏覽歷史的上一頁與下一頁。

window.history.back();   // 上一頁
window.history.forward();   // 下一頁


我們也可以透過 go() 方法，令網頁移動到特定的歷史記錄，例如：

window.history.go(-1);    // 上一頁
window.history.go(1);    // 下一頁
window.history.go(0);    // 等同重新整理當前頁面

在 HTML5 中還加入了 history.pushState() 和 history.replaceState() 方法，可用來加入或修改歷史記錄，這裡僅先作拋磚引玉，留到未來有機會再深入研究。


location：

location 可對網頁的 URL 網址進行操作，像是取得目前頁面的網址，或是將瀏覽器導向指定的頁面。以下我們直接看範例：

See the Pen js-learn: location-1 by Trong (@itrong) on CodePen.

location.href 可返回目前網頁的 URL 值，我們也可以指定網址給它，讓使用者點擊指定元素時將瀏覽器導向指定網址，例如：

<button onClick="window.location.href='https://www.google.com.tw/'" Value="Click">點我</button>

除了上述的 href，還有這些屬性方法可以使用：

用途
hostname
回傳網頁主機的域名
pathname
回傳目前頁面的路徑與文件名稱
port
回傳網頁主機的端口
protocol
回傳網頁使用的傳輸協定（http: 或 https:）

另外要提的是 location.assign 與 location.replace，location.assign 相當於 HTML 中的 <a> 元素，可直接令網頁連接到指定的 URL，因此可以點擊上一頁返回前一個歷史記錄；而 location.replace 則是將指定的 URL 替換目前網頁的內容，即使點擊上一頁也無法返回前一個瀏覽的記錄。

See the Pen js-learn: location-2 by Trong (@itrong) on CodePen.
