前言

jQuery 的選擇器（Selector）某種意義上就類似 CSS 取得 DOM 元素的方式，如果你對 CSS 有一定認識，那麼當你繼續閱讀內文 jQuery 的選擇器列表時，就會發現 jQuery 選擇器都和 CSS 十分雷同。而本篇除了依據選擇器的類型進行分類外，也會對 jQuery 選擇器的使用建議稍作補充說明。

指定次序

選擇器
說明
備註
$(selector:first)
第一個元素
$(selector:last)
最後一個元素
$(selector:not(:first))
除了第一個以外的全部元素
$(selector:not(:last))
除了最後一個以外的全部元素
$(selector:odd)
奇數元素
$(selector:even)
偶數元素
$(selector:eq(n))
指定索引值
n 從 0 開始起計，代表第一個指定元素，n 為 1 即代表第二個指定元素，依此類推。


除了在選擇器裡使用 CSS selector 概念外，jQuery 本身也有 .first()、.last()、.eq() 等方法可以直接指定特定數字的元素進行操作。

// 指定第一個元素
$(selector).first()

// 指定最後一個元素
$(selector).last()

// 指定索引值為 3 的元素
$(selector).eq(2)

這些方法稱之為「遍歷」（Traversing Methods），之後的文章會再作較詳細的介紹。

屬性值

指定選擇器的屬性值公式為 $(selector[attribute = value])，例如：

$(div[class="name"])

此外，在選擇器指定元素的屬性後方添加特定符號，還能追加更細部的條件，以下我們以 <a> 元素為例：

選擇器
說明
備註
$(a[href^="string"])
選取指定連結「開頭」為指定字串的項目
$(a[href$="string"])
選取指定連結「結尾」為指定字串的項目
$(a[href*="string"])
選取指定連結「包含」指定字串的項目

另外，也可以指定表單元素的狀態，像是 disabled、checked...等，例如：

$("input:enabled")
$("input:disabled")
$("input:checked")


補充

儘管 jQuery 提供開發者非常便捷的選擇器直觀易懂地去選取指定元素，但其效能問題在高階開發環境中則被視為相當重要的議題，一派開發者認為，過多條件的選擇器將會造成網頁運作的負擔。因此，在使用 jQuery 選擇器之前，不妨先考慮以下幾點事項：

  1. 利用 ID 選擇器取代其他選擇器
  2. 盡量少用 Class 選擇器
  3. 減少不必要的選擇器
  4. 盡量使用 Javascript 原生選取方法

前三項淺顯易懂故不再多提，關於第四項，以下簡單舉一個例子：

var item = document.getElementById("demoID");
$(item).css("color", "#F00");
