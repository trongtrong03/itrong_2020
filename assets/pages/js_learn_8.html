「JavaScript 是個『弱型別』的程式語言」──這是剛接觸 JavaScript 常聽到的其中一句金言，意即弱型別程式語言的變數本身不具有資料型別，它（變數）的「值」才有資料型別的資訊。順便再簡單提強型別程式語言的殊異，有別於弱型別，強型別程式語言定義的變數在宣告時期就必須指定該變數的型別是什麼，如果該變數執行了不是它型別的運算，程式就會出現非預期的問題。

複習過強、弱型別程式語言的概念，接著再來重新溫習 JavaScript 定義的資料型別分別有哪些：

這是先前變數一文做的資料型別表格，事實上，JavaScript 將這些資料型更上一層歸類成兩類：Primitive（基本型別）與 Object（物件型別）。

Primitive（基本型別）：

In JavaScript, a primitive (primitive value, primitive data type) is data that is not an object and has no methods.

MDN web docs

根據 MDN 的解釋，基本型別的值是除了物件之外，原始定義的純值。它囊括以下六種類型：Number、String、Boolean、Null、Undefined 與 Symbol。


Object（物件型別）：

物件型別只有一種型別，那就是物件（Object），它是一個由 Name 與 Value 組成（又稱鍵值配對）的集合體，一個集合體即是物件的「屬性」（Property）。此外，物件型別還有「次型別」（subtype），例如函式物件、陣列物件等。

var obj = {
    // 函式物件，又稱物件方法
    Meth: function(){
        document.write('Hello world!') ;
    },

    // 陣列物件
    Arr: ['apple', 'orange', 'banana'],
}
