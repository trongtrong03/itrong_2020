前言

在 <div> 佈局概念還未普及之前，開發者們習慣用 <table> 進行網頁排版，但受限表格欄列式並排的排版結構，很難實現或必須用更多的表格嵌套實現較繁瑣的版型設計，遂後來 <div> 逐漸取代 <table>，成為迄今依舊蔚為主流的排版佈局。儘管 <div> 大大提升網頁排版的靈活度，但是在部分需求上仍顯得支絀──例如令元素垂直居中對齊且等高，排版必須使用不下於 <table> 階層數量的 <div> 才能達到需求。例如：

基本概念

CSS Flexible Box Layout，一般以 Flexbox 稱之，它是一種彈性容器（盒子），可以更簡單有效解決複雜的網頁佈局，誠如前言描述的垂直居中效果，如果使用 Flexbox 便可以大大減少 <div> 的使用。Flexbox 的基本結構是這樣子的：

父容器

display：

父容器的 display 參數必須設置為 flex 或 inline-flex，Flexbox 才有作用。

.flexbox {
    display: flex;
}

flex 與 inline-flex 在於前者排列模式等同 block，會獨自佔據一整列，後續元素將會排在其下方；而 inline-flex 則近似 inline-block，後續的元素會與之並排。且看範例演示：

See the Pen CSS3: flexbox-1 by Trong (@itrong) on CodePen.


flex-flow：

此屬性是 flex-direction 與 flex-wrap 的合體簡寫屬性，前者用來控制子元件排列的主軸方向，後者則是設定當子元件溢出父容器寬度範圍時是否要換行。

.flexbox {
    flex-flow: flex-direction flex-wrap;
}

flex-direction：

參數值
說明
row
【預設值】由左至右，從上到下
row-reverse
由右至左，從下到上
column
從上到下，再由左至右
column-reverse
從下到上，再由右至左

此屬性設定的排列方向為「主軸」，而與之垂直的另一軸則稱為「交錯軸」，它將會直接影響 justify-content 與 align-items 對齊依據，一般而言，若 flex-direction 設定是水平排列，則 justify-content 將會是水平方向對齊方式（主軸為水平），align-items 則設定垂直方向的對齊（交錯軸為垂直）。
